# Engineer

**Gameplay: Deployable Management**

Camera: third person. Input: placement-based abilities, device management UI. Core loop: deploy, position, manage, redeploy.

Flux usage: moderate. Powers deployables.

| Spec | Identity | Playstyle |
|---|---|---|
| Architect | Static deployables | Turrets, shield generators, healing stations, barricades. Territorial control |
| Pilot | Active drone control | Controls a single powerful drone, can switch to drone-cam. Almost two characters |
| Saboteur | Offensive tech | EMP, hacking enemy shields, disruption fields, overloading enemy equipment |

All three specs share a simplified visual scripting system for device behavior. Before fights, players wire together condition/action blocks for their deployables: "if enemy health below 30%, switch target." Looks and feels like programming without requiring programming skill. A quick "debug" mini-game triggers when devices malfunction mid-combat.

TTRPG source classes: Technologue, Taliste (partial)
